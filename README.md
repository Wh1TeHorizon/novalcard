# 小说角色卡提取器 v5.6

把一整本小说丢进去，自动扒出角色、世界观、文风，一键打包成酒馆（SillyTavern）能直接导入的完整角色卡。纯前端单文件，一个 `.html` 双击就能用，不装环境、不搭后端。

---

## 能做什么

- 角色卡：从小说提取角色，按模板生成结构化人设卡；支持批量、生成后微调、截断自动续写、详细程度可选、按人物阶段拆分。
- 世界书：提取世界观 / 势力 / 地点 / 设定等条目；可加共享背景大纲、配角群像（合成一条或每人一条）、剧情分段总结、人物列表、剧情大纲；支持按意见修改 / 继续输出。
- 文风蒸馏：分析全书写作风格，产出可直接粘给 AI 的「文风提示词」。
- 一键打包：角色卡 + 世界书 + 文风打包成一张完整角色卡，导入酒馆即一并到位。
- user 设置：把「你」写进角色的世界（设定与角色的关系，或让 {{user}} 替换书中某角色）。
- 自检：对照原文检查已生成内容有没有 OOC、经历互串、错误，给出问题日志和修正稿。
- 模板管理（五套内置）、多套 API 配置、自定义提示词、历史记录、小说记录、断点续跑、一键停止、报错自查。

---

## 用法（从上到下走一遍）

1. 接口设置：选接口格式（OpenAI 兼容 / Gemini 原生），填 Base URL、API Key、模型名，可「拉取模型」下拉选。可存多套配置切换。
2. 导入小说：拖 txt 进框或粘贴正文。编码自动识别（UTF-8 / GBK / UTF-16）。
3. 选择角色：知道要谁就直接手动输名字（免扫描）；不确定就「扫描全书」后点选。同名不同昵称可「合并所选」；需要的话可勾「按人物阶段拆分」。
4. 生成角色卡：选模板、选详细程度、点选角色，生成。可微调、续写。
5. （可选）世界书：一键提取，可加共享背景、配角群像、剧情分段、人物列表、剧情大纲，给世界书取名。
6. （可选）文风蒸馏：一键分析。
7. 打包导出：点「导出完整角色卡 .json」，角色卡 + 世界书 + 文风打包成一张卡，导入酒馆。
8. （可选）自检：扫描当前内容，勾选要查的项，一键检查。

## 在线版：更新即刷新（推荐）

本工具支持部署成一个网址，之后作者更新，用户刷新就是最新版，不用每次重新下载文件。页面右上角会显示版本号（如 v5.6），方便确认自己是不是最新。

部署办法（任选其一，均免费）：

- GitHub Pages：把 html 传到 GitHub 仓库（建议改名为 `index.html`），Settings → Pages → 从 main 分支根目录部署，得到一个 `https://用户名.github.io/仓库名/` 网址。
- Cloudflare Pages / Vercel / Netlify：连上 GitHub 仓库自动部署，域名更干净。

注意：在线版是 https。若使用者填的是本地 `http://127.0.0.1` 接口，会被浏览器拦（混合内容）；在线版更适合用远程 https 中转的人。用本地接口的人，仍可下载 html 到本地打开使用。

## 世界书能做的条目

- 提取世界书：世界观 / 势力 / 地点 / 设定 / 历史 / 物品，核心世界观常驻（蓝灯），具体条目关键词触发（绿灯）。
- 共享背景大纲（蓝灯）：多角色共享的时代背景、共同经历。
- 配角处理：合成一条「群像合集」，或每个配角单独一条绿灯条目；人数上限可调（默认 20）。
- 剧情分段总结（绿灯）：按阶段 / 副本拆分，每段一条（不少于 300 字，含出场角色、前因后果、道具、时间线）。
- 人物列表（蓝灯）、剧情大纲（蓝灯）：始终注入的常驻总纲。
- user 替换：把某角色名替换成 {{user}}。

## 五套模板

- 默认模板：完整字段的标准人设卡。
- 事件导向模板：把经历拆成关键事件、转折点、人物关系。
- 精简人设（无经历）：配合共享背景大纲用，个人卡只留差异。
- NPC 模板（配角简版）：身份、性格、与主角关系、剧情作用。
- 空白模板（自定义）：只留骨架，从零自定义字段。

## 常问的几点

- 要不要先扫描再生成才准？不是。扫描只是帮你列出有哪些角色供点选，不影响生成质量。知道要谁就直接手动输名字，一样准、更省。
- 「全文 12 万字」是啥？只在勾了「用全文」时才是截前 12 万字（长篇不建议勾）。默认走「相关段落」模式，跨全书均匀采样、覆盖开头到结尾。两个上限都能在高级参数调。
- 撞限流 / 中途停了？扫描、世界书、批量角色卡都支持断点续跑——再点一次从断点继续，已完成的不重做。

## 隐私

全部在本机浏览器里跑。小说内容只发给你自己填的接口做分析，不经过任何第三方。所有设置、模板、历史、小说记录都存在本机浏览器——清除浏览器数据会一并清掉，重要成果记得下载保存。

---

## 常见问题

- Load failed / Failed to fetch：请求没到服务器，多是浏览器拦截。http 接口别在 https 页面调；接口需支持跨域；本地接口确认在运行。
- 429 / 限流：等一会再试，或换限流更宽的模型；已内置自动重试。
- 只输出思考、没正文（含文风显示完成却空白）：思考型模型把 token 用光了，调大 max_tokens 或换 flash 系模型。
- 卡在处理中：接口静默卡住约 90 秒会自动中止；也可点高亮按钮上的「停止」手动中断。
- 导入乱码：工具会自动识别 UTF-8 / GBK / UTF-16；仍乱码就把文件另存为 UTF-8 再导入。
- 版权拒绝：部分模型对长篇有版权限制，换更宽松的模型来跑小说提取即可。

工具内置「报错自查」一节，遇到问题可对照排查。

## 使用提醒

模型抽取难免有偏差，生成后建议对照原文核对一遍再用（也可用内置「自检」）。工具只负责把原文里的信息整理成卡，不替原文补设定。
